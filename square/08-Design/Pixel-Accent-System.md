# Pixel Accent System

> 01 Log Square에서 픽셀을 어디에 쓰고 어디에는 쓰지 않을지 정의하는 시각 규칙 문서.

---

## 목표

- 탑다운 픽셀 감성으로 오피스 정체성을 만든다.
- 가독성과 업무 툴 사용성을 해치지 않는다.
- 귀여움은 포인트로만 사용하고, 본문 인터페이스는 현대적으로 유지한다.

## 레퍼런스 감각

- 옛 탑다운 RPG 스타일
- 1타일 캐릭터
- 작은 오브젝트와 조형물
- 방마다 다른 소품과 상태 변화

중요한 점은 `감각은 참고하되 그대로 모사하지 않는다`는 것이다.

## 픽셀 사용 위치

### 사용한다

- 프로젝트 홈 또는 맵 보기의 오피스 도면
- 방 아이콘과 방 썸네일
- 대표 캐릭터
- 에이전트 1타일 아바타
- 상태 오브젝트
  - 경고등
  - 체크보드
  - 모니터 glow
  - 문패 변화
- 빈 상태 화면
- 작은 애니메이션 포인트

### 사용하지 않는다

- 긴 텍스트 영역
- 메시지 본문 타이포그래피
- 문서 읽기 화면
- 복잡한 폼
- 설정 페이지 전체

## 대표 캐릭터 규칙

- 맵 안에 항상 존재한다.
- 현재 집중 중인 방에 위치한다.
- 직접 이동 조작은 필수가 아니다.
- 대표는 게임 플레이어 캐릭터보다 `오피스의 앵커`로 보이게 한다.

## 에이전트 오브젝트 예시

### COO

- 체크리스트 보드
- 정돈된 파일함

### CSO

- 화이트보드
- 카드 스택

### CTO

- 모니터
- 서버 랙 느낌의 오브젝트

### Risk Critic

- 경고등
- 위험 표지판

### Verifier

- 체크보드
- 승인 스탬프

### Documentation Manager

- 서류함
- 아카이브 상자

## 애니메이션 규칙

- 느리고 과장되지 않게
- 상태 변화를 읽게 하는 수준으로만
- idle motion은 최소화
- 대표 승인, red flag, summary ready 같은 의미 있는 순간에만 강조

## 위험 요소

- 전체 UI가 게임처럼 보이는 것
- 픽셀이 본문 정보 밀도를 방해하는 것
- 귀여움이 책임 구조보다 앞서는 것
- 픽셀 자산이 너무 많아 일관성이 깨지는 것

## 디자인 문장

01 Log Square의 픽셀은 게임 플레이를 위한 것이 아니라, 오피스 공간의 존재감과 상태를 보여주는 `작은 생명감`이다.
