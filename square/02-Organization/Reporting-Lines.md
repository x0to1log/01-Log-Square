# Reporting Lines

> 기능형 계층 구조를 기본으로 한다. 대표 아래에 전략 코어와 검수 코어가 있고, 실행/기록/스페셜리스트는 각 책임 라인 아래에 배치된다.

---

## 구조

- `대표`
- `COO 운영 이사`
- `CSO 전략 기획자`
- `CTO 기술 수석`
- `Risk Critic`
- `Verifier`
- `도큐멘테이션 매니저` → COO 산하
- `Builder / Implementation Lead` → CTO 산하
- `트렌드 스카우트` → CSO 산하
- `콘텐츠 크리에이터` → CSO 산하
- `브랜드 디자이너` → CSO 산하, 단 브랜드 핵심 안건은 대표 승인

## 보고 라인

- COO는 대표에게 프로젝트 진행 상황, 일정, 우선순위를 보고한다.
- CSO는 대표에게 방향성, 사업 가설, 포지셔닝, 외부 메시지 방향을 보고한다.
- CTO는 대표에게 기술적 가능성, 구조, 구현 리스크를 보고한다.
- Risk Critic은 대표에게 직접 리스크와 반대 의견을 보고한다.
- Verifier는 대표에게 직접 검증 상태, 미충족 조건, 승인 전 체크 결과를 보고한다.
- 도큐멘테이션 매니저는 COO에게 회의록과 SOP 상태를 보고한다.
- Builder는 CTO에게 구현 초안, 프로토타입, 실행 설계 상태를 보고한다.
- 트렌드 스카우트는 CSO에게 외부 신호와 동향을 보고한다.
- 콘텐츠 크리에이터는 CSO에게 메시지 초안과 발신물을 보고한다.
- 브랜드 디자이너는 CSO에게 비주얼/감각 방향을 보고하고, 중요 안건은 대표 확인을 받는다.

## 왜 이렇게 두는가

- COO는 운영 실무의 허브다.
- CSO는 외부 세계와 의미 체계의 허브다.
- CTO는 기술 판단의 허브다.
- Risk Critic과 Verifier는 독립성을 잃지 않기 위해 대표 직속으로 다룬다.
- Builder는 구현 품질을 위해 CTO 라인에 두는 것이 자연스럽다.
